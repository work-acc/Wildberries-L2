Что выведет программа? Объяснить вывод программы. Объяснить как работают defer’ы и их порядок вызовов.

```go
package main

import (
	"fmt"
)


func test() (x int) {
	defer func() {
		x++
	}()
	x = 1
	return
}


func anotherTest() int {
	var x int
	defer func() {
		x++
	}()
	x = 1
	return x
}


func main() {
	fmt.Println(test())
	fmt.Println(anotherTest())
}
```

Ответ:
```
2
1

Функция test() возвращает значение переменной x, которая увеличивается на 1 
внутри отложенной функции с помощью defer. Однако отложенная функция выполняется 
после того, как x была присвоена значению 1 в строке x = 1, поэтому x увеличивается 
на 1 после того, как она была присвоена. Поэтому функция test() возвращает значение 2.

Функция anotherTest() также увеличивает значение переменной x на 1 внутри отложенной 
функции с помощью defer. Однако в этом случае x возвращается из функции до вызова 
отложенной функции, поэтому она не влияет на возвращаемое значение. Поэтому функция 
anotherTest() возвращает значение 1.

Теперь об объяснении работы отложенных функций (defer):

Отложенные функции в Go выполняются после завершения выполнения функции, 
в которой они были объявлены, но до того, как функция возвращает результат.
Они выполняются в порядке обратном их объявлению, то есть последняя объявленная 
отложенная функция будет выполнена первой, а первая - последней.
Важно отметить, что отложенные функции имеют доступ к значениям переменных 
на момент их объявления, а не на момент их выполнения.

```
